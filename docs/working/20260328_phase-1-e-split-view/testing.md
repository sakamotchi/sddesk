# テスト手順書 - Phase 1-E: 右ペイン分割表示（Split View）

## 概要

Phase 1-E（Split View）のテスト手順を記載します。タブモードと Split モードの切り替え、Split 時のレイアウト・操作を手動で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること

---

## 手動テスト

### ケース 1: Split ボタンでモード切り替え

**手順:**

1. アプリを起動する
2. タブバー右端の `Columns2` アイコンボタンをクリックする
3. もう一度クリックする

**期待結果:**

- 1回目クリック: Split モードに切り替わり、左にコンテンツ・右にターミナルが横並びで表示される
- Split ボタンがアクセントカラーでハイライトされる
- 2回目クリック: タブモードに戻る

**確認結果:**

- [x] OK

---

### ケース 2: `Ctrl+\` ショートカットで切り替え

**手順:**

1. タブモードの状態で `Ctrl+\` を押す
2. Split モードの状態でもう一度 `Ctrl+\` を押す

**期待結果:**

- 1回目: Split モードに切り替わる
- 2回目: タブモードに戻る

**確認結果:**

- [x] OK

---

### ケース 3: Split モードのレイアウト確認

**手順:**

1. Split モードに切り替える
2. 各ペインのヘッダーバーを確認する
3. ペイン間のセパレーターをドラッグしてリサイズする

**期待結果:**

- 左ペインに「📄 コンテンツ」ヘッダーと `[ ✕ ]` ボタン、右ペインに「> ターミナル」ヘッダーと `[ ✕ ]` ボタンが表示される
- セパレーターをドラッグして左右の幅を変更できる

**確認結果:**

- [x] OK

---

### ケース 4: `[ ✕ ]` ボタンで Split 解除

**手順:**

1. Split モードに切り替える
2. 左ペイン（コンテンツ側）の `[ ✕ ]` ボタンをクリックする
3. 再度 Split モードにして、右ペイン（ターミナル側）の `[ ✕ ]` ボタンをクリックする

**期待結果:**

- どちらの `[ ✕ ]` を押してもタブモードに戻る
- タブモードに戻った後、`activeMainTab` は変化しない（直前のタブが表示される）

**確認結果:**

- [x] OK

---

### ケース 5: Split モード中の `Ctrl+Tab` 無効化

**手順:**

1. Split モードに切り替える
2. `Ctrl+Tab` を押す

**期待結果:**

- Split モード中は `Ctrl+Tab` でのタブ切り替えが発生しない

**確認結果:**

- [x] OK

---

### ケース 6: Split モードでターミナルが操作できる

**手順:**

1. Split モードに切り替える
2. 右ペインのターミナルにコマンドを入力する（例: `echo "hello"`）

**期待結果:**

- ターミナルに入力でき、コマンドが実行される
- ターミナルタブの追加・切り替えも動作する

**確認結果:**

- [x] OK

---

### ケース 7: Split モードの状態が再起動後も復元される

**手順:**

1. Split モードに切り替える
2. アプリを終了して再起動する

**期待結果:**

- 再起動後も Split モードが復元される（`mainLayout` は persist 対象）

**確認結果:**

- [x] OK
