# テスト手順書 - Phase 3-C: キーボードショートカット整備

## 概要

キーボードショートカット機能（タブ操作・分割切り替え・ヘルプモーダル）の手動テスト手順を記載します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- プロジェクトフォルダが開かれていること

---

## 手動テスト

### ケース 1: ターミナルタブを新規作成する（コンテンツペインから）

**手順:**

1. コンテンツペインを表示する
2. `Cmd+T` を押す

**期待結果:**

- ターミナルペインに切り替わり、新しいターミナルタブが追加される
- 新しいシェルセッションが起動する

**確認結果:**

- [x] OK

---

### ケース 2: ターミナルタブを新規作成する（ターミナル内から）

**手順:**

1. ターミナルペインを表示し、シェルが起動していることを確認する
2. `Cmd+T` を押す

**期待結果:**

- ターミナルタブが1つ追加され、アクティブになる
- 新しいシェルセッションが起動する

**確認結果:**

- [x] OK

---

### ケース 3: タブを閉じる

**手順:**

1. コンテンツタブが2枚以上ある状態にする（ケース1で追加）
2. `Cmd+W` を押す

**期待結果:**

- アクティブなタブが閉じる
- 左隣のタブがアクティブになる

**確認結果:**

- [x] OK

---

### ケース 4: タブが1枚のみの場合 `Cmd+W` が無効になる

**手順:**

1. コンテンツタブが1枚のみの状態にする
2. `Cmd+W` を押す

**期待結果:**

- タブが閉じない（無効化されている）

**確認結果:**

- [x] OK

---

### ケース 5: `Cmd+1`〜`Cmd+9` でタブ切り替え

**手順:**

1. コンテンツタブを3枚作成する
2. `Cmd+1` を押す
3. `Cmd+3` を押す

**期待結果:**

- `Cmd+1` で1番目のタブがアクティブになる
- `Cmd+3` で3番目のタブがアクティブになる

**確認結果:**

- [x] OK

---

### ケース 6: `Ctrl+Shift+Tab` で前のタブへ移動

**手順:**

1. コンテンツタブを複数作成し、2番目以降をアクティブにする
2. `Ctrl+Shift+Tab` を押す

**期待結果:**

- 1つ前のタブがアクティブになる

**確認結果:**

- [x] OK

---

### ケース 7: コンテンツペインを分割する

**手順:**

1. コンテンツペインを表示する
2. `Cmd+\` を押す

**期待結果:**

- コンテンツペインが左右（または上下）に分割される

**確認結果:**

- [x] OK

---

### ケース 8: コンテンツペインの分割を解除する

**手順:**

1. コンテンツペインが分割されている状態で `Cmd+\` を押す

**期待結果:**

- 分割が解除され、1ペインに戻る

**確認結果:**

- [x] OK

---

### ケース 9: ターミナルペインを分割する

**手順:**

1. ターミナルペインを表示する
2. `Cmd+Shift+\` を押す

**期待結果:**

- ターミナルペインが分割される

**確認結果:**

- [x] OK

---

### ケース 10: `Cmd+0` でツリーパネルへフォーカス移動

**手順:**

1. コンテンツペインにフォーカスがある状態で `Cmd+0` を押す

**期待結果:**

- ツリーパネルにフォーカスが移動する

**確認結果:**

- [x] OK

---

### ケース 11: `?` キーでショートカット一覧を表示する

**手順:**

1. ターミナル・テキスト入力欄以外にフォーカスがある状態で `?` を押す

**期待結果:**

- ショートカット一覧モーダルが開く
- カテゴリ別にショートカットが一覧表示される
- キーがバッジ形式（`⌘ T` 等）で表示される

**確認結果:**

- [x] OK

---

### ケース 12: `Esc` でショートカット一覧を閉じる

**手順:**

1. ショートカット一覧モーダルを開く
2. `Esc` を押す

**期待結果:**

- モーダルが閉じる

**確認結果:**

- [x] OK

---

### ケース 13: ターミナル入力中にショートカットが誤発動しない

**手順:**

1. ターミナルタブを開き、シェルプロンプトが表示されていることを確認する
2. ターミナル内で `?` を入力する
3. ターミナル内で `t` を入力する（`Cmd+T` ではなく普通の文字）

**期待結果:**

- `?` がターミナルに入力されるだけで、ヘルプモーダルは開かない
- `t` がターミナルに入力されるだけで、タブが新規作成されない

**確認結果:**

- [x] OK

---

### ケース 14: 既存ショートカットが引き続き動作する

**手順:**

1. `Ctrl+Tab` を押す
2. `Ctrl+P` を押す
3. `Esc` を押す

**期待結果:**

- `Ctrl+Tab` でコンテンツ↔ターミナルペインが切り替わる
- `Ctrl+P` でパス検索パレットが開く
- `Esc` でパレットが閉じる

**確認結果:**

- [x] OK
