# テスト手順書 - Claude Code スラッシュコマンドサジェスト統合 Phase A

## 概要

Phase A（組み込みコマンド + バンドル Skill 静的リスト）のテスト手順書。可能な限り手動操作で確認する。自動テストは合成関数・UI 描画に関わる部分に限定する。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- テスト対象のプロジェクトフォルダが準備されていること
- **既存**のユーザー定義プロンプトテンプレートが 1 件以上存在するとベター（衝突確認のため）。なければテスト前に `⌘T` → 新規作成で `review-pr` / body `/review pull/{{number}}` を追加する

## 手動テスト

### ケース 1: コマンドセクションが表示される

**手順:**

1. アプリを起動する
2. ターミナルタブがアクティブな状態で `Cmd+Shift+P`（mac）/ `Ctrl+Shift+P`（Win/Linux）でパレットを開く
3. textarea に `/` を 1 文字だけ入力する

**期待結果:**

- `Commands` セクションに `resume` / `clear` / `model` など 20 件以上の組み込みコマンドが表示される
- 各行の左端に `CMD` バッジが表示される
- 各行に英語の `description` が表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 2: テンプレートと同時表示

**手順:**

1. ケース 1 の続きで、`/` だけの状態を維持
2. セクションの並び順を確認

**期待結果:**

- `Commands` セクションの下に `Templates` セクションがある
- `Templates` セクションにユーザー定義テンプレ（`review-pr` 等）が表示される
- 各テンプレ行の左端に `TPL` バッジが表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 3: fuzzy 検索で横断マッチ

**手順:**

1. パレット開いた状態で textarea に `/rev` と入力

**期待結果:**

- `Commands` セクションに `review` / `rewind` が絞り込まれる
- `Templates` セクションに `review-pr`（既存のもの）が絞り込まれる
- 他の無関係コマンド（`clear` 等）は表示されない

**確認結果:**

- [ ] OK / NG

---

### ケース 4: 空セクション非表示

**手順:**

1. 現在登録されているテンプレに **マッチしない** クエリを入力（例: `/xyzzy` など存在しない名前）

**期待結果:**

- `Templates` セクションが非表示になる（見出しも出ない）
- `Commands` セクションも候補 0 の場合は非表示
- 全セクションが空になる場合は `SlashSuggest` 全体が非表示

**確認結果:**

- [ ] OK / NG

---

### ケース 5: キーボード操作（↑/↓ セクション境界）

**手順:**

1. `/` だけ入力した状態で `↓` キーを連打
2. コマンドセクションの最終行を超えたところで `↓` を押す

**期待結果:**

- activeIndex のハイライトがコマンド最終行から `Templates` セクションの先頭行に移動する
- さらに `↓` で templates セクション内を移動
- `Templates` 最終行で `↓` を押しても動かない（リスト末尾）
- `↑` で逆方向に同様に動く

**確認結果:**

- [ ] OK / NG

---

### ケース 6: builtin 選択時の `/name ` 挿入

**手順:**

1. `/` を入力してコマンドセクションを表示
2. `↓` で `resume` に合わせる
3. `Enter` を押す

**期待結果:**

- textarea の内容が `/resume `（末尾スペースあり）に置換される
- textarea にフォーカスが残る
- PTY には**送信されない**（ターミナル表示に何も出ない）

**確認結果:**

- [ ] OK / NG

---

### ケース 7: template 選択時の既存挙動（回帰）

**手順:**

1. `/review-p` 等と入力してテンプレを絞り込み
2. `↓` でテンプレ行に合わせる
3. `Enter` を押す

**期待結果:**

- draft がテンプレ body に全置換される
- `{{number}}` 等のプレースホルダがあれば最初のプレースホルダが `setSelectionRange` で選択状態になる
- 送信はされない

**確認結果:**

- [ ] OK / NG

---

### ケース 8: IME 変換中は誤挿入しない

**手順:**

1. `/` を入力してコマンドセクション表示
2. 日本語 IME をオンにして `あ` 等を入力中（未確定状態で `Enter` を押す）

**期待結果:**

- `Enter` は IME 確定に使われ、候補選択は**されない**
- 未確定状態が続いているあいだ、サジェストは表示されたまま

**確認結果:**

- [ ] OK / NG

### ケース 9: クリック選択

**手順:**

1. `/` を入力
2. マウスで適当なコマンド行にホバー → クリック

**期待結果:**

- クリックで `/<name> ` が挿入される

**確認結果:**

- [ ] OK / NG

---

### ケース 10: 送信挙動（⌘Enter）

**手順:**

1. ケース 6 で `/resume ` が挿入された状態
2. `⌘Enter`（mac）/ `Ctrl+Enter`（Win/Linux）を押す

**期待結果:**

- `/resume \n` が PTY に書き込まれ、Claude Code セッションに実際に `/resume` が実行される
- パレットが閉じる
- 履歴に `/resume` が追加される

**確認結果:**

- [ ] OK / NG
