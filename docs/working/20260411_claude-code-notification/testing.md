# テスト手順書 - claude-code-notification

## 概要

OSC 9 シーケンスによる自動通知と、通知設定 UI の動作を確認します。可能な限り手動操作で確認します。

## 前提条件

- `npx tauri dev` でアプリが起動していること
- macOS の通知権限を SpecPrompt に許可していること（初回起動時に権限ダイアログが出る）
- `claude` コマンドがインストールされていること

---

## 手動テスト

### ケース 1: Claude Code 作業完了時に通知が届く

**手順:**

1. SpecPrompt のターミナルで `claude` を起動する
2. 何らかのタスクを依頼して Claude Code に実行させる
3. Claude Code の作業が完了するまで別アプリに切り替えて待つ

**期待結果:**

- macOS のデスクトップ通知が表示される
- タイトルが `SpecPrompt / Claude Code`、本文が Claude Code のメッセージ内容である

**確認結果:**

- [ ] OK / NG

---

### ケース 2: OSC 9 シーケンスを手動で送信して通知をテストする

**手順:**

1. SpecPrompt のターミナルで以下のコマンドを実行する
   ```bash
   printf '\e]9;テスト通知\x07'
   ```
2. macOS 通知センターを確認する

**期待結果:**

- タイトル `SpecPrompt / Claude Code`、本文 `テスト通知` の通知が表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 3: 通知 OFF に設定すると通知が発火しない

**手順:**

1. アプリ内設定画面を開く
2. 「Claude Code 通知」トグルを OFF にする
3. ケース 2 と同じ OSC 9 コマンドを実行する
4. macOS 通知センターを確認する

**期待結果:**

- 通知が表示されない

**確認結果:**

- [ ] OK / NG

---

### ケース 4: 通知設定がアプリ再起動後も維持される

**手順:**

1. アプリ内設定画面で「Claude Code 通知」を OFF にする
2. アプリを終了する
3. アプリを再起動する
4. 設定画面を開く

**期待結果:**

- 「Claude Code 通知」トグルが OFF のまま維持されている

**確認結果:**

- [ ] OK / NG

---

### ケース 5: 通知 ON に戻すと再び通知が届く

**手順:**

1. ケース 3 の続きから、「Claude Code 通知」トグルを ON に戻す
2. ケース 2 と同じ OSC 9 コマンドを実行する

**期待結果:**

- 通知が再び表示される

**確認結果:**

- [ ] OK / NG

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| macOS 通知権限を拒否した状態で OSC 9 を受け取る | 通知は表示されないがアプリはクラッシュしない | |
| 複数ターミナルタブが開いている状態で OSC 9 を受け取る | 対象タブからの通知のみ発火する | |
| OSC 9 の本文が空文字の場合 | 本文なしで通知が表示される（または発火しない） | |
